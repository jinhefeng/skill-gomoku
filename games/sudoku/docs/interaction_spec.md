# 交互说明书: 双人对战数独 (Sudoku)

## 1. 视觉设计
### 1.1 棋盘布局 (Sudoku Board)
- **栅格系统**：9x9 标准栅格，每个 3x3 宫格（Block）用较粗的边框 (2px) 区分。
- **响应式尺寸**：棋盘容器 `width: min(88vw, 45vh); aspect-ratio: 1/1;`
- **数字渲染**：居中对齐，大字号为已填入数字，小字号 (1-9 九宫格分布) 为草稿。

### 1.2 背景与高亮
- **普通模式**：棋盘深色主题背景。
- **草稿模式**：整盘背景叠加半透明暖色，提示用户当前为草稿操作态。
- **选中格高亮**：当前选中的格子高亮显示。
- **错误反馈**：填错后对应格子底色变为 `rgba(255,0,0,0.08)` 持续 800ms。

### 1.3 玩家得分面板
- **复用平台公共面板**：数独渲染器不向 `p1GameArea`/`p2GameArea` 注入额外的得分区域。得分仅通过平台公共的 `#p1Score`/`#p2Score` 展示，避免重复显示。

### 1.4 棋盘尺寸优化
- **桌面端**：棋盘容器 `width: min(520px, 88vw, 60vh); aspect-ratio: 1/1;`（较之前 400px 提升 30%）以充分利用大屏空间。
- **移动端**：保持 `min(88vw, 50vh)` 不变。

## 2. 大厅难度选择
### 2.1 入口位置
- 难度选择 UI 内嵌在大厅的数独「游戏卡片」底部。
- 以三个按钮呈现：**简单** / **中等** / **困难**，默认选中「中等」。

### 2.2 交互流程
1. 玩家点击数独卡片后，卡片展开显示难度选项。
2. 选定难度后，以 `player_select_game({ gameId: 'sudoku', difficulty })` 发射到服务端。
3. 服务端缓存首位选中 sudoku 的玩家的 `difficulty`；后选玩家点击数独卡片时直接接受该难度。
4. 双方共识达成后自动开始游戏。

## 3. 游戏内交互逻辑
### 3.1 填入流程
1. **选中格**：点击/触碰某一空格，该格高亮。
2. **输入数字**：通过屏显数字键盘 (1-9) 输入。
3. **提交逻辑**：
   - 正式模式→发送 `game_move` 至后端。
   - 草稿模式→本地 toggle 该格对应数字草稿状态。

### 3.2 草稿切换按钮
- 位于数字键盘旁，带铅笔图标 (✏️) + "草稿" 文案。
- 切换时整体棋盘背景色有明显视觉变化。

### 3.3 擦除按钮
- 提供一个擦除键 (⌫) 用于清除当前选中格的草稿。

## 4. 音效定义
| 动作 | 音色 | 音量 |
| :--- | :--- | :--- |
| 我方填对 | 清脆叮当 (ding) | 100% |
| 我方填错 | 低沉错误 (buzz) | 100% |
| 模式切换 | 轻微点击 (click) | 60% |
| 对手填对 | 短促轻音 (tink) | 40% |

## 5. 移动端适配
- 不调用系统键盘，使用游戏内 1-9 数字面板。
- 数字面板位于棋盘下方。
- 棋盘格不低于 35px×35px。
