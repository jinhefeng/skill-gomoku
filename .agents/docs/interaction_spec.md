# 交互说明书 (Interaction Spec) - Game Platform

## 1. 游戏大厅交互 (Lobby UI)
- 只要没有进行中的游戏，屏幕正中心大舞台强制呈现**巨大的“选择游戏版面 (lobbySelector)”**，而非被覆盖的影子节点。
- **动态选边渲染**：点击游玩项不再出 Modal 弹窗，而是通过 CSS Highlight 发光外边框直接标记己方卡片。同时，卡片中会印出醒目的指示副标：“已就绪 (你)”与对方同步可见的“已就绪 (对手)”。
- 当双排标记完全合并为同时选择了同款游戏时，选单无需经由任何动画干涉，立刻抹除进入实际的棋盘舞台。

## 2. 侧边底栏规范
- **Player Panels**：分局界面左右。在 HTML 层面上极其克制，仅只给出 `#p1GameArea` 等无内容的空白锚点盒子。<br>倒计时秒表、能量环这类极其侵入的内容不再硬写入 `index.html`。由各游戏的专属 `XXXRenderer` 占据它并画出它的样子。
- **大厅弹幕组件**：常驻于屏幕底端中心（脱离了棋盘物理包围），无论在什么环境下都可以在两人间传递信息。但在开局时，底部的开源链接（GitHub Footer）应当退位让出空间。

---

## 3. 游戏退出与战败流 (Immediate Surrender)
- 在交战中，位于头部中央的 **[返回大厅]** 是**紧急止损按钮**。
- **发起方体验**：若中途点击该按钮，立刻放弃本局（视为认输），画面中的游戏棋盘瞬间被强制没收、抹除所有的本地线程计时器并归还空白的大厅游玩卡片选单模型。
- **滞留方（对手）体验**：立刻收到一并抛出的强制宣告（结算 Overlay）：提示“你赢了”。出于对手已走不可互交的逻辑，该面板上的【再来一局】系统级按钮将会被隐藏。面板只剩一个寂寥的【返回大厅】，点击后该滞留方棋盘才被顺带销毁。

---

## 4. 全局异常处理指示 (Global Flow)
- **断线拦截 Alert机制**：大厅对决系统需要保证双方信息差对称，如果对方在任何非游戏时段关闭了浏览器离开，强制调起不可避开的系统 `alert()` 告诉本方对战已经破裂。并借此阻断事件循环直到用户强行点击确认，随后执行页面完全 `reload`。
- 其他无伤大雅的消息提示继续采用深色毛玻璃样式的自绘 Toast 从天而降并在 2.5 秒后自行湮灭。
